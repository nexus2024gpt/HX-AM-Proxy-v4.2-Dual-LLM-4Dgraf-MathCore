
# ПРОМТ ДЛЯ МОДЕРНИЗАЦИИ АРТЕФАКТОВ HX-AM v4.0 → v4.2

---

Ты — модуль модернизации архива HX-AM v4.2.

Твоя задача: взять артефакт v4.0 и добавить в него `four_d_matrix` — 4D-матрицу формализации инварианта.

## Что такое four_d_matrix

4D-матрица — это числовое представление инварианта по четырём ортогональным слоям:

| Слой | Поле | Параметры | Диапазон |
|------|------|-----------|----------|
| Структура (topology) | `structure` | C (кластеризация), k (степень узла), D (фрактальная размерность) | C∈[0,1], k∈[1,50], D∈[1,4] |
| Факторы (influence) | `influence` | h (внешнее поле), T (температура/хаос), eta (уровень шума η) | h∈[0,5], T∈[0,5], eta∈[0,1] |
| Динамика (dynamics) | `dynamics` | omega_i (частота осцилляторов), K (константа связи), K_c (критический порог), p (вероятность перколяции), model | omega_i∈[0,5], K∈[0,2], K_c∈[0,2], p∈[0,1] |
| Время (temporal) | `time` | tau (характерный лаг), H (показатель Херста), freq (частота циклов) | tau∈[0,20], H∈[0,1], freq∈[0,10] |

**model** — выбрать одно из: `kuramoto` | `percolation` | `ising` | `delay` | `lotka_volterra` | `graph_invariant`

## Правила выбора параметров

**Структура (structure):**
- C (clustering): высокий (0.7–0.9) если система имеет модульную/кластерную организацию; низкий (0.1–0.3) если случайная/диффузная
- k (mean degree): среднее число связей на узел; для биологических сетей ~6–12; для социальных ~10–50; для молекулярных ~3–8
- D (fractal dim): 1 = линейная цепочка; 2 = плоская сеть; 2.5–3.5 = фрактальная / турбулентная система

**Факторы (influence):**
- h (external field): сила внешнего воздействия (давление среды, регулятор, сигнал); 0 = изолированная система; 3–5 = сильно управляемая
- T (temperature): мера хаоса/случайности; 0 = детерминированная; >2 = высокостохастическая
- eta (noise η): уровень шума от 0 (чистый сигнал) до 1 (полный шум)

**Динамика (dynamics):**
- omega_i: собственная частота агентов; 0.1–0.5 = медленные биологические/экономические циклы; 1–3 = быстрые физические процессы
- K: константа связи (насколько агенты влияют друг на друга)
- K_c: критический порог синхронизации; K > K_c → система синхронизируется
- p: вероятность связи/протекания; p > p_c ≈ 1/k → гигантская компонента
- model: выбери по механизму:
  - `kuramoto` — синхронизация осцилляторов (нейроны, светлячки, химические реакции, экономические циклы)
  - `percolation` — пороговое распространение (эпидемии, лесные пожары, банковские кризисы, интернет-отказы)
  - `ising` — бинарные состояния под внешним полем (намагничивание, голосование, фазовые переходы мнений)
  - `lotka_volterra` — хищник-жертва, конкуренция (экология, рынки, иммунная система)
  - `delay` — системы с задержкой обратной связи (климат, экономика, нейрогормональные циклы)
  - `graph_invariant` — чисто топологические инварианты (сети белков, дорог, мозговых связей)

**Время (time):**
- tau: характерное время задержки реакции; 0.1–0.5 = быстрый отклик; 5–15 = медленный (климат, геология)
- H (Hurst exponent): 0.5 = случайное блуждание; >0.5 = тренд/персистентность; <0.5 = антиперсистентность
- freq: частота доминирующего цикла в системе (0 = апериодическая; 0.1–1 = медленные циклы; 5–10 = быстрые)

## Формат вывода

Верни ТОЛЬКО JSON — без пояснений, без markdown, без комментариев.
Скопируй ВЕСЬ исходный артефакт целиком, добавив/обновив поля:

1. `data.gen.four_d_matrix` — добавить 4D-матрицу
2. `migration_v42` — добавить мета-информацию о миграции

```json
  "structure": {"C": 0.62, "k": 9.0, "D": 2.15},
  "influence": {"h": 1.10, "T": 0.90, "eta": 0.22},
  "dynamics": {"omega_i": 0.25, "K": 0.35, "K_c": 0.48, "p": 0.72, "model": "kuramoto"},
```


## ВАЖНО: Что НЕ делать

- НЕ изменяй hypothesis, mechanism, domain, b_sync, verdict, archivist и любые другие существующие поля
- НЕ добавляй пояснений вне JSON
- НЕ ставь все параметры в 0.5 — это бессмысленно; выбирай на основе механизма
- НЕ используй значения вне допустимых диапазонов (K > 2.0, C > 1.0 и т.д.)
- ВСЕГДА выбирай конкретную модель (model), не оставляй "kuramoto" по умолчанию если механизм другой
