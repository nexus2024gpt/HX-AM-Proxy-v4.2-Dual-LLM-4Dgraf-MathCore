## 📊 Что такое MGAP?

**MGAP (Metric GAP)** — система переноса числовых инвариантов из артефактов HX-AM в прикладные отраслевые модели.

### Полный цикл для одного артефакта:
1. Загрузка артефакта + извлечение 4D-матрицы и порогов симуляции
2. Ранжирование моделей реестра по compute_4d_resonance (13-мерный вектор)
3. Перевод параметров τ, K, η на язык отрасли (translation_map)
4. Генерация кода адаптации под целевую программу (диспетч по math_type)
5. Расчёт на синтетических данных (example_data из реестра)
6. Формирование вывода для разработчика и бизнеса

### v4.4 исправления vs пилот:
- ✅ Правильное извлечение: tau → four_d["time"]["tau"], K → four_d["dynamics"]["K"], eta → four_d["influence"]["eta"]
- ✅ Резонанс через compute_4d_resonance() (13-мерный вектор из FourDMatrix)
- ✅ Top-K матчей, отсортированных по резонансу (не только первый по math_type)
- ✅ Диспетч кода по math_type: graph_invariant / kuramoto / delay
- ✅ LLM вызывается с правильной сигнатурой llm.generate(prompt)
- ✅ Все 11 моделей реестра имеют полные поля

**Логия:** Логистика

- Влияние: h=1.2, T=0.8, η=0.35
- Динамика: K=0.55, K_c=0.45, ω_i=0.15, p=0.68
- Время: τ=3.2, H=0.65, freq=0.8

**Перевод параметров:**
- τ → Lead time поставщика (дни): 1–30 дней
- K → Коэффициент реакции на дефицит (0=нет реакции, 1=мгновенное пополнение)
- η → Коэффициент вариации спроса CV = σ/μ: 0.1–1.0

**Слепая зона (критические пороги):**
Стандартные алгоритмы WMS рассчитывают страховой запас линейно от CV и L, игнорируя нелинейное взаимодействие: при CV>{eta_max} и L>{tau_max} дней стандартная формула занижает SS в 1.2–1.5 раза, провоцируя каскадный дефицит.

**Формула адаптации:**
```
multiplier = max(1.0, (CV / eta_crit) * (L / tau_crit))
SS_new = SS_old * multiplier
```

## 📝 Примечания

1. **4D-резонанс:** Вычисляется через compute_4d_resonance() — 13-мерный вектор изоморфизм между артефактом и моделью
2. **Весовые коэффициенты:** 70% резонанс + 30% совпадение math_type
3. **Порядок приоритетов:** перевод → верификация (стресс-тест) → critical thresholds → simulation → model critical
4. **LLM опционально:** Улучшение шаблонов слепых зон через llm.generate()
